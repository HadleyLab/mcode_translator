# Strict Infrastructure, Fail-Fast, No Implicit Fallbacks
- **No silent defaults**: every configuration must be **explicit**. If a setting is missing or invalid, **raise immediately**.
- **Fail fast**: prefer raising precise exceptions at the point of failure over retrying or returning sentinel values.
- **No hidden retries**: retries must be **explicit**, bounded, and instrumented; default is **no retries**.
- **Explicit timeouts**: all I/O and external calls must specify tight, documented timeouts.
- **No global mutable state**: initialization occurs in a single, explicit composition root; pass dependencies via constructors.
- **Strict parsing/validation**: use `pydantic`/dataclasses validators; reject partial/ambiguous inputs.
- **Deterministic startup**: validate environment and configuration at startup; abort on any missing/invalid values.
- **Performance-first**: prefer lean, allocation-conscious code; avoid convenience helpers that add layers unless justified.
- **Observability**: structured logs, counters/timers for critical paths; treat warnings as actionable (fail in CI if new warnings appear).
- **Contracts over convenience**: prefer exact types, preconditions/postconditions, and invariant checks with clear exceptions.
