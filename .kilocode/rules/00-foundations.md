# Foundations
- Prefer **clarity over cleverness**. Small, composable functions; explicit names; single responsibility.
- Propose the **minimal change** that solves the problem; do not break public interfaces without an explicit migration plan.
- Any behavior change must update tests, docs, and examples in the same PR.
- If requirements are ambiguous, state assumptions and propose 2–3 concrete options with trade-offs.
